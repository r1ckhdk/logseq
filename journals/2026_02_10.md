- [[RNP - Administração de Sistemas Linux]]
	- **Módulo 3 - Sistema de arquivos**
		- Estrutura dos discos
			- Cada disco é dividido em uma ou mais partições
			- Uma partição deve ser formatada com um sistema de arquivos conhecido pelo SO
			- É necessário ter uma partição montada no diretório raíz (/)
			- Cada partição compõe uma subárvore, cuja raiz é o seu ponto de montagem
			- Todas essas partições estão acessíveis em uma árvore de diretórios única
		- Particionamento
			- Geralmente é definido durante a instalação do sistema
			- Vantagens:
				- Isolamento de falha
			- Estruturas de Particionamento
				- MBR: Master Boot Record
					- Primária
					- Estendida
					- Lógica
					- Comando `fdisk` é utilizado para criar uma tabela de partições do tipo MBR
						- `# fdisk /dev/sda`
				- GPT: Guid Partition Table
					- Suporte até 128 partições primárias
					- Compatível com o formato MBR
					- Permite a identificaçao das partições através de labels ou UUID
					- Limite de 1ZB para tamanho de arquivos
					- Compatívei com a interface UEFI de inicialização
				- LVM: Logical Volume Management
					- Camada de software que permita ao administrador gerenciar diversos dispositivos diferentes de tal forma que possam ser organizados para formar novos dispositivos de armazenamento.
						- Otimiza a utilização do espaço em disco
						- Identifica as partições por um nome
						- Permite o redimensionamento das partições
						- Novos dispositivos são adicionados de forma transparente
						- É possível mover dados de um volume físico on-the-fly
						- Suporte a espelhamento
				- Sistema de arquivos
					- Representa a organização lógica de uma partição, ele define como os arquivos são armazenados e acessados no disco.
					- Sistema de arquivos ext4, xfs, ntfs etc. -> Virtual File System (VFS)
					- Journaling
						- Escrita no sistema de arquivos > Journal > disco
