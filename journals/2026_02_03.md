	- **Módulo 1**: Introdução ao sistema operacional Linux
		- **Arquitetura [[Linux]]**
			- Organizada em camadas
				- Hardware -> Kernel -> Libs -> Aplicações / Shell -> Usuário
			- Kernel
				- Núcleo do OS
				- Responsável por prover interface entre hardware e aplicações
				- Supervisor, autorizando o uso dos recursos entre os processos
			- Bibliotecas de funções
				- Chamadas feitas por processos ao SO para acessar recursos
				- Uso de funções padrão (*open, close, read, write etc.*)
			- Shell
				- Lê e interpreta comandos de entrada de um terminal
				- Cria novos processos à medida que são requisitados
				- *sh, csh, bash, ksh, tcsh etc*
		-
		- **Linux Standard Base (LSB)**
			- Contribui para interoperabilidade entre as distribuições
			- Define uma base padrão para os sistemas baseados em Linux
			- Composta de duas partes:
				- Uma comum a todas as distribuições
				- Outra específica para cada arquitetura de processador
				-
			- Atualmente está na versão 5.0
			-
			- **Filesystem Hierarchy Standard (FHS)**
			-
		- **Sistema de arquivos**
			- Inode
				- Grande tabela onde as informações de metadados para cada arquivo
					- Tipo, dono, grupo, permissões
					- Datas de criação e modificação
					- Links
					- Tamanho
					- Endereço do arquivo no disco rígido
		- **Tipos de Arquivos**
			- Named pipe
				- Permite a comunicação entre dois processos rodando em uma mesma máquina
				- É criado com o comando `mknod` ou `mkfifo`
				- É removido com o comando `rm`
				- Interliga processos que não possuem relação entre si
		- Permissões de arquivos
			- Bits especiais
				- Sticky bit
					- Evita mecanismo de swap-on / off em arquivos executáveis (em desuso)
					- Controla a escrita em diretórios de uso geral
					- Representado pela letra `t`
				- Setgid / Setuid
					- Usados em arquivos executáveis e diretórios
					- Representado pela letra `s`
